# loglint
Линтер для логов на go, совместимый с golangci-lint.

# Описание сущностей
## Analyzer
Здесь поисходит анализ файлов в директории, построение ast дерева, и сбор LogMessage для последующей обработки

На эту сущность приходится больше всего тестов, так как это главная часть программы. Для тестирования здесь использовался фреймворк golang.org/x/tools/go/analysis/analysistest

## Detector
Эта сущность распознает логи. Сейчас сделана не в общем виде, так как в условии требовались конкретные логи

## LogMessage
Абстракция над объектом сообщения, он сделан "на будущее", так как если код придется расширять он обязательно потребуется

## Extractor
Достает сообщение. Постарался написать его "умным", он парсит и конкатенацию и функцию Sprintf

Здесь я принял несколько условностей, так как в условии об этом не было сказано:
- правило lowercase считает пустой лог ошибкой

# Настройка и запуск
Для настройки нужно отредактировать следующие настройки в .golangci.yml
```yml
check-lowercase: true
check-english: true
check-specialchars: true
check-sensitive: true
sensitive-case: false
sensitive-keywords:
- password
- token
- secret
```

Флаги отвечают за включение правила, sensitive-keywords задает список чувствительных слов, а с помощью sensitive-case можно указать будут ли чувствительные слова чувствительны к регистру

После настройки нужно собрать библиотеку
```bash
golangci-lint custom
```

После чего появится файл **gcl-with-loglint** (я уже сделал такой для ubuntu). Возможно потребуется сделать его исполняемым с помощью chmod. Использование:

```bash
./gcl-with-loglint run ./gcl-with-loglint run ./testdata
```
